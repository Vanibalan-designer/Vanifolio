The Challenge

Bridging Design & Engineering

At Doctor Anywhere, we had a comprehensive design system called Gravity built in Figma. While Figma served as our source of truth for design, engineers often struggled to translate designs into code efficiently.

The gap between design specifications and production code was causing delays, inconsistencies, and frustration across teams. Engineers needed a functional documentation where they could directly access and copy code snippets.

The Goal

Build a Storybook documentation from scratch that would serve as a living, functional reference for engineers ‚Äî complementing Figma as the design source of truth while providing production-ready code for implementation.

Key Objectives

  • Functional code documentation
  • Support white label portal
  • Cross-platform compatibility
  • Cost-effective solution
  • Easy maintenance

The Process

Trying Multiple Methods

Building a Storybook from scratch wasn't straightforward. I explored multiple approaches to find the most efficient and cost-effective solution that would work for our team.

Using Cursor AI as My Co-Pilot

I leveraged Cursor AI to accelerate the development process. As a designer building something technical, Cursor helped me write functional code, debug issues, and implement best practices ‚Äî all while learning along the way.

  • AI-assisted code generation
  • Real-time debugging support
  • Learning while building
  • Rapid iteration cycles
Building with Cursor AI

Figma as Source of Truth

While Storybook became the code reference, Figma remained our design source of truth. I ensured that every component in Storybook matched its Figma counterpart exactly ‚Äî creating a seamless bridge between design and development.

  • 1:1 mapping with Figma components
  • Design token synchronization
  • Visual consistency checks
  • Documentation alignment
Figma to Storybook Workflow

Cross-Team Collaboration

Working with cross-functional teams helped us find cost-saving methods for launching and hosting the Storybook. We ultimately deployed it on Bitbucket's static hosting, which was both free and integrated with our existing workflow.

  • Engineering team input on code structure
  • DevOps support for deployment
  • Cost-effective Bitbucket hosting
  • Integrated CI/CD pipeline

Key Learnings

Designers Can Build

With AI tools like Cursor, designers can create functional technical solutions without being full-stack developers. It's about problem-solving, not just coding.

Cross-Team Value

Collaborating with engineering and DevOps early helped find cost-effective solutions and ensured the final product actually met their needs.

Documentation is Product

Good documentation is a product in itself. It needs UX thinking, iteration, and user feedback ‚Äî just like any other product we design.

Start Scrappy, Iterate

Trying multiple methods before finding the right approach is part of the process. The key is to start, learn, and iterate quickly.
